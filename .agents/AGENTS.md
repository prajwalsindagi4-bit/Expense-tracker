# Rules

## Error Logging Protocol
Whenever asked to store or log an error, always document it in the `known_errors_and_fixes.md` file located at the root of the project. Use the following strict format for every entry:

- **The Error**: [Describe what went wrong]
- **The Flow**: [Describe the sequence of events that led to the error]
- **The Solution**: [Describe how the error was solved]
- **The Impact**: [Describe what was fixed by applying the solution]
- **The Use Case**: [Explain why this fix matters and how it is used]
